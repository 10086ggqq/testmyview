## 项目简介

「拾光记」是一款面向 **需要记忆大量简答题的学习者** 而开发的纯前端单页应用。

它提供了一种 **低成本、无依赖、零部署** 的记忆训练方案：把你的题目粘贴到文本框 → 一键解析 → 进入背题 / 记忆 / 实战 任意模块开始训练。所有数据通过 `localStorage` 保存在浏览器本地，不经过任何服务器，不上传任何云端。

内置了一份 **计算机视觉题库** 作为示例，开箱即可体验。

## 功能特性

| 模块 | 说明 |
|------|------|
| 🏠 **首页** | 题库概览 + 快速导入区域，粘贴文本一键解析 |
| ☰ **背题** | 按顺序浏览题目 → 答案 → 解析，支持上一题 / 下一题 / 选题弹窗 |
| ✎ **记忆** | 先背再输入，系统自动判断对错并给出相似度；悬停 👁 查看标准答案 |
| ⚔ **实战** | 随机抽题，即时判分，进度条实时展示正确率 |
| ◉ **我的** | 累计作答 / 正确次数 / 正确率，支持一键清空 |

### 交互亮点

- **拾光风格视觉**：粗黑边框、高对比配色、错位卡片、夸张阴影，视觉冲击强
- **底部导航自动隐藏**：页面静止 2 秒后收起，鼠标移动立即出现
- **点击按压动效**：按钮按下时向下偏移 4px，阴影消失，模拟物理按压
- **响应式布局**：桌面、平板、手机均适配良好
- **多题库管理**：可同时导入多个题库并自由切换

## 题库格式

在首页文本框中粘贴以下格式的文本，点击「导入题库」即可解析：

```
#name: 题库名称
#id: bank_unique_id

[type:essay]
题目内容在这里
[answer:标准答案内容]
[explain:可选的解析说明内容]

[type:essay]
第二题的题目内容
[answer:第二题的答案]
[explain:第二题的解析]
```

### 字段说明

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| `#name:` | 是 | 题库显示名称 |
| `#id:` | 是 | 题库唯一 ID（英文 / 数字 / 下划线） |
| `[type:essay]` | 是 | 标记一道新题目开始 |
| 题目文本 | 是 | 紧跟在 `[type:essay]` 后的第一行内容 |
| `[answer:...]` | 是 | 标准答案 |
| `[explain:...]` | 否 | 解析说明，可选 |

### 示例

```
#name: 计算机视觉题库
#id: cv_questions

[type:essay]
计算机视觉的常见任务有哪些？
[answer:图像分类、目标检测、图像分割、人脸识别、姿态估计、图像生成等。]
[explain:计算机视觉是让机器理解图像和视频的学科，常见任务涵盖从基础识别到高级生成。]

[type:essay]
OpenCV 的核心模块是什么？主要包括哪些函数？
[answer:核心模块是 cv2，主要包括图像读取、显示、保存、变换、滤波、特征检测等函数。]
[explain:OpenCV 的 Python 接口为 cv2，提供了丰富的图像处理函数。]
```

## 判题逻辑

应用内置了一个 **关键词匹配判题器**，工作流程如下：

1. **预处理**：去除用户输入和标准答案的标点（，。！？、；：""''（）()【】）、统一空格、转小写
2. **分词**：按空格切分标准答案为关键词数组
3. **匹配**：统计用户答案中命中的关键词数量
4. **计算**：`相似度 = 命中数 / 关键词数`
5. **判定**：
   - 记忆模式：`相似度 ≥ 0.8` → 正确
   - 实战模式：`相似度 ≥ 0.5` → 正确

> ⚠️ 简答题判题本身是一个 NLP 难题，本算法适用于 **关键词导向** 的答案。如果你的答案依赖语序或复杂语义，建议把答案改写为以关键词 / 短语为主的形式。

### 核心文件职责

- **[index.html](file:///e:/Textnote/QuestionBank/Essay/index.html)**：5 个页面（首页 / 背题 / 记忆 / 实战 / 我的） + 底部导航的静态结构，无第三方依赖
- **[styles.css](file:///e:/Textnote/QuestionBank/Essay/styles.css)**：定义了拾光风格主题变量（`--red` `--yellow` `--border` `--shadow` 等），所有组件样式，以及 640px / 380px 两档响应式断点
- **[app.js](file:///e:/Textnote/QuestionBank/Essay/app.js)**：
  - `parseBankText()`：解析题库文本 → 结构化题目对象
  - `compareAnswer()`：判题核心算法
  - `render*()`：各页面渲染函数
  - `state` / `stats`：应用状态（当前题库、题目索引、训练统计）
  - `localStorage`：读写题库数据与学习进度

## 本地开发

无需安装任何依赖，直接编辑文件即可。推荐开发流程：

```bash
# 1. 克隆仓库
git clone https://github.com/your-username/essay.git
cd essay

# 2. 启动本地 HTTP 服务（避免 file:// 协议的某些限制）
python -m http.server 8000

# 3. 浏览器打开 http://localhost:8000

# 4. 修改 index.html / styles.css / app.js，刷新浏览器查看效果
```

### 调试小贴士

- 在浏览器开发者工具 → Application → Local Storage 中，可以看到两个 key：
  - `brutal-memory-bank`：题库数据
  - `brutal-memory-stats`：学习统计数据
- 删除这两个 key 即可重置应用状态

## 部署到 GitHub Pages

1. 将本目录推送到 GitHub 仓库的 `main` 分支
2. 进入仓库 **Settings → Pages**
3. Source 选择 `Deploy from a branch`，Branch 选择 `main`，目录选择 `/ (root)`
4. 保存后等待 1-2 分钟，访问 `https://your-username.github.io/essay/`

或者使用更简单的方式：**直接用浏览器打开 index.html**，这是一个零依赖的静态应用。
